## 搬瓦工测速的几种方法，先搞懂工具

在聊各个机房之前，先说说搬瓦工测速到底怎么做，因为方法用错了，测出来的结果毫无参考价值。

**方法一：直接 ping 测试 IP**

搬瓦工官方为每个机房提供了专用测试 IP，可以在购买前直接 ping 来测延迟。比如：

- DC9 CN2 GIA 洛杉矶机房测试 IP：`89.208.246.192`
- DC6 CN2 GIA-E 洛杉矶机房测试 IP：`173.82.219.132`
- 香港 CN2 GIA 机房测试 IP：`93.179.124.145`
- 日本东京 CN2 GIA 机房测试 IP：`163.170.134.5`
- 日本大阪软银机房测试 IP：`185.212.58.138`

Windows 打开命令提示符，输入 `ping 89.208.246.192` 就能看到延迟。Mac 直接在终端里同样操作。

想看全国各地延迟的话，可以把 IP 丢进 `ping.chinaz.com` 这类在线工具，一次性看到三大运营商在全国主要城市的延迟分布，比本地 ping 更有参考价值。

**方法二：下载测速文件**

看延迟只是第一步，实际下载速度才是更直观的感受。搬瓦工官方提供了各机房的测速文件（100MB/1GB 大小），直接下载观察速度。晚高峰时段（晚上8点到12点）下载一次，白天再下一次，两个数据对比就能知道这个机房抗压能力如何。

**方法三：SpeedTest 演示站点**

这是最直观的方式。搬瓦工中文网和一些第三方站长搭建了演示站点，进去之后可以直接跑 Speedtest 测上传下载速度，还能用 LookingGlass 功能跑 traceroute 查看路由路径，看数据包经过了哪些节点、是否走 CN2 线路。

**方法四：一键测速脚本**

已经买了VPS、登录进去之后，可以在服务器端跑 `superspeed.sh` 脚本，测试从这台VPS到全国各地节点的下载速度，结果比本地测更全面，涵盖电信、联通、移动三网。
